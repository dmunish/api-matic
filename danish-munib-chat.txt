  Steps:
  1. The script prints an authorization URL.
  2. Open it in a browser, sign in with the Google account whose calendar you
  want to poll, click Allow.
  3. Google redirects to http://localhost:5000/oauth/callback?code=...&.... The
  page will fail to load (no server running on :5000) ‚ÄĒ that's expected. Copy
  the code value from the URL bar.
  4. Paste it back into the prompt.

  That writes google_token.json to the project root.

  Before you run it, make sure these env vars are set (in .env):
  - GOOGLE_OAUTH_CLIENT_ID
  - GOOGLE_OAUTH_CLIENT_SECRET
  - GOOGLE_OAUTH_REDIRECT_URI=http://localhost:5000/oauth/callback

  The redirect URI in .env must match exactly what's registered in your Google
  Cloud Console OAuth client. If you haven't set up an OAuth client yet, you'll
  need to do that first at console.cloud.google.com ‚Üí APIs & Services ‚Üí
  Credentials ‚Üí Create OAuth Client ID (type: Web application) and add the
  redirect URI there.

  Once the token is saved, re-run pytest and
  test_get_recently_ended_meetings_live will hit your real calendar.
